## Our promise (and the honest limits)

<Aside type="tip" title="What we guarantee">
	Between your device and Anthropic, **no middle step can read or store your prompt** — including us, the ApexOne operator.
</Aside>

<Aside type="caution" title="What we don't overstate">

- **Anthropic sees your plaintext.** The Claude route connects to a Claude subscription, so the request must ultimately reach Anthropic in the clear, handled under Anthropic's own privacy policy. We never claim confidentiality _from_ Anthropic.
- We **do not claim** "your model was not swapped" — judge by the actual replies you receive and by the verifiable evidence below.
- Internally, the "gateway → Meridian" step crosses the network as plaintext (protected by the encrypted connection and an access token), but **both ends sit inside hardware-protected secure computers**.

</Aside>

We spell out the scope precisely so that the part we _can_ prove is genuinely trustworthy.

## How to verify it yourself

<p><Badge text="≈ 30 seconds" variant="success" size="medium" /></p>

<Steps>

1. Open [**apex1.us/proof**](https://apex1.us/proof).

2. The page **automatically** starts verifying in your browser — you'll see several checks go from "Checking" to a green "Passed".

3. All green means: **genuine hardware + running the public, untampered code + a proof generated right now** (not an old, replayed one).

</Steps>

The whole process runs **in your own browser, with no reliance on our servers** — it asks chip-maker **Intel's root of trust** to confirm "this is genuine hardware, running that public code." Click **Re-verify** anytime; the system uses a fresh one-time random number and checks again.

### What each check proves

| What you'll see | Plain meaning |
| --- | --- |
| Genuine Intel secure hardware | Not an ordinary server pretending — a real chip with hardware-level isolation. |
| Code fingerprint matches | It runs exactly the code we publish, not a single byte changed. |
| Bound to a one-time random number | This proof was generated **right now**, so nobody can fool you with an old report. |
| Signed receipt | This request is signed by the authenticated secure computer and can be checked later. |

## Check it independently

Don't want to just take our word for it? You don't have to. The [**apex1.us/proof**](https://apex1.us/proof) page publishes the exact reference values (the "fingerprints") for the current deployment and lets you — or any independent third party — compare them, line by line, against the hardware proof read live. **If the values match, the running code really is the public, untampered code we claim.**

Everything you need is on the `/proof` page itself:

- the **current reference values** for each secure computer (App IDs, deployment fingerprints, source version, container-image identities);
- **direct links** to the public source code and to the signed container images;
- a one-click **download of the proof report**, which you can re-check in an independent explorer such as [proof.t16z.com](https://proof.t16z.com/);
- verification that runs against **Intel's own attestation data**, not ours.

<Aside type="note">
	We keep the exact fingerprint values on the `/proof` page rather than in this guide on purpose: the **verification method** described here never changes, but the specific fingerprints _do_ change every time we ship an update — and the page always shows the current ones. This guide stays valid; the page stays live.
</Aside>

## FAQ

### Does the automatic verification use my data or API key?

No. The verification that runs when the page opens is **read-only and free** — it sends none of your content and needs no API key. Your API key is used only when you **actively click** the end-to-end encryption test (and even then it stays only in your browser).

### What if verification shows "incomplete" or "failed"?

Click **Re-verify** and try again first — occasional network issues (e.g. the Intel verification-data service being temporarily unreachable) can cause "incomplete." If it keeps failing, **do not trust** that report and [contact us](mailto:support@apex1.us).

### Does this guarantee Anthropic won't see my content?

**No**, and we never claim it does. The Claude route must hand plaintext to Anthropic. What we guarantee is that every middle step between your device and Anthropic cannot read or keep your content.

### Do I need to understand cryptography to verify?

No. Just open `/proof` and look for the green checks. The terminology and reference values are there for advanced users who want to dig deeper.
