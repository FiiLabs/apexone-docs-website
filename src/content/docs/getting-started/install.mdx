## Install Node.js

We recommend the latest **LTS (Long-Term Support)** release. ApexOne requires **Node.js 22.x or later**.

<Tabs syncKey="os">
	<TabItem label="Windows">

	<Steps>

	1. Visit [nodejs.org](https://nodejs.org).
	2. Download the latest **LTS** installer.
	3. Run the downloaded `.msi` file and follow the installation wizard.
	4. Restart your terminal before continuing.

	</Steps>

	</TabItem>
	<TabItem label="macOS">

	**Recommended: NVM** — [NVM](https://github.com/nvm-sh/nvm) lets you manage multiple Node.js versions and switch between them per project.

	```sh
	# Install NVM
	curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.5/install.sh | bash

	# Reload your shell configuration
	source ~/.zshrc

	# Install the latest LTS release
	nvm install --lts

	# Set it as the default version
	nvm alias default 'lts/*'

	# Use it now
	nvm use --lts
	```

	**Alternative: Homebrew**

	```sh
	brew install node
	```

	</TabItem>
	<TabItem label="Linux">

	**Recommended: NVM**

	```sh
	# Install NVM
	curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.5/install.sh | bash

	# Reload your shell configuration
	source ~/.bashrc

	# Install and use the latest LTS release
	nvm install --lts
	nvm alias default 'lts/*'
	nvm use --lts
	```

	**Alternative: package manager**

	```sh
	# Ubuntu / Debian
	sudo apt update && sudo apt install -y nodejs npm

	# Fedora / RHEL / CentOS
	sudo dnf install -y nodejs
	```

	<Aside type="caution">
		Distribution packages are often several major versions behind. If `node --version` reports
		anything older than 22, use NVM (above) or [NodeSource](https://github.com/nodesource/distributions)
		to get a current release.
	</Aside>

	</TabItem>
</Tabs>

### Verify Node.js

Open a terminal and run:

```sh
node --version
npm --version
```

Example output:

```
v22.14.0
10.9.2
```

If both commands return version numbers, Node.js has been installed successfully.

## Install Git

Git is a distributed version control system used by Claude Code and most modern development workflows.

<Tabs syncKey="os">
	<TabItem label="Windows">

	<Steps>

	1. Visit [git-scm.com/download/win](https://git-scm.com/download/win).
	2. Download and run the latest installer.
	3. During setup, we recommend:
	   - **Git from the command line and also from 3rd-party software**
	   - **Use Windows' default console window**
	4. Complete the installation using the default settings.

	</Steps>

	</TabItem>
	<TabItem label="macOS">

	**Option 1 — Homebrew**

	```sh
	brew install git
	```

	**Option 2 — Xcode Command Line Tools**

	```sh
	xcode-select --install
	```

	**Option 3 — Official installer:** download from [git-scm.com/download/mac](https://git-scm.com/download/mac).

	</TabItem>
	<TabItem label="Linux">

	```sh
	# Ubuntu / Debian
	sudo apt update && sudo apt install -y git

	# Fedora / RHEL / CentOS
	sudo dnf install -y git
	```

	</TabItem>
</Tabs>

### Verify Git

```sh
git --version
```

Example output:

```
git version 2.43.0
```

If a version number is displayed, Git has been installed successfully.

### Configure Git

After installation, configure your Git identity:

```sh
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
```

To verify your configuration:

```sh
git config --list
```

<Aside type="note">
	Your Git username and email appear in commit history. Use the identity associated with your
	development workflow.
</Aside>

## Troubleshooting

<Aside type="caution" title="Node.js installation failed (Windows)">
	If the installer fails: run it as **Administrator**, temporarily disable antivirus software, and
	verify that sufficient disk space is available.
</Aside>

**`npm` command not found**

- **Windows** — verify that the Node.js installation directory has been added to your system `PATH`.
- **macOS / Linux** — restart your terminal or reload your shell configuration:

  ```sh
  source ~/.zshrc   # or ~/.bashrc
  ```

**Git displays incorrect characters**

If Git shows non-English filenames incorrectly:

```sh
git config --global core.quotepath false
```
